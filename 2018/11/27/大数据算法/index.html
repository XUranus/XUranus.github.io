大数据算法概述

定义

大数据算法指的是，在给定的资源约束下，以大数据为输入，在给定时间约束内可以生成满足给定约束结果的计算。

大数据算法可以不是

  • 精确算法
  • 内存算法
  • 串行算法
  • 仅在电子计算机上运行的算法（可以有人来参与到程序的执行）

大数据不仅仅是：

  • 云计算
  • MapReduce
  • 大数据分析与挖掘的算法

难度

  • 访问全部数据时间过长
    • 读取部分数据 -> 时间亚线性算法
  • 数据难于放入内存计算
    • 将数据存储到磁盘上 -> 外存算法
    • 仅基于少量数据计算 -> 空间亚线性算法
  • 单个计算机难以保存全部数据，计算需要整体数据
    • 并行处理 -> 并行算法
  • 计算机知识不足
    • 人来参与 -> 众包算法

算法设计

  • 精确算法
  • 并行算法
  • 近似算法
  • 随机算法
  • 在线算法/数据流算法
  • 外存算法
  • 面向新型体系结构的算法
  • 现代优化算法

亚线性算法

时间/空间/能量/通讯/IO消耗是O(输入规模)

  1. 亚线性时间算法

    • 亚线性时间近似算法：例：给一个社交网络，能否在不访问所有节点的情况下，求平均每个节点的度数
    • 性质检测算法

  2. 亚线性空间算法

    • 例：有限的内存空间，面对一个无限数据流，求中位数（如何不保存所有数字，求中位数）

水库抽样—-一个亚线性空间算法

输入：一组数据，大小未知
输出：这组数据的k个均匀抽样
要求：

  1. 仅扫描一次
  2. 空间复杂度O(k)
  3. 扫描到数据前n个数字时候(n>k),保存当前已扫描数据的k个均匀抽样

算法：

  1. 申请一个长度为k的数组A保存抽样
  2. 保存首先接收到的k个元素
  3. 当接收到第i个新元素的t时，以k/i的概率随机替换A中的元素(即生成[1,i]间的随机数j,若j<=k,则以t替换A[j])

性质

  1. 采样是均匀的
  2. 空间复杂度O(k)

平面图直径计算—-时间亚线性算法

输入：m个顶点的平面图，任意两点之间的距离存储在矩阵D中，即点到点的距离是D[i,j]

  • 输入大小是:

  • 最大的D[i,j]是图的直径
  • 点之间的距离对称且满足三角不等式

输出：图的直径
要求：运行时间为O(n)

无法在要求的时间内得到精确解，寻找近似算法

近似算法：

  1. 任意选择k<=m
  2. 选择似的D[k,l]最大的l
  3. 输出D[k,l]和(k,l)

精确解D,D[k,l]<=D<=2*D[k,l]

全0数组判定—-亚线性时间判定算法

输入：包含n个元素的0,1数组A
输出：A中的元素是否全是0

要求：运行时间为O(n)

判定问题的近似解的定义：输入满足某种性质或者远非满足此性质

ξ远离：对于输入x，如果从x到L中任意字符串的汉明距离至少为ξ|x|,则x是ξ-远离L的

算法：

  1. 在A中随机独立抽取s=2/ξ个位置上的元素
  2. 检查抽样，若不包含1，则输出是，否则输出否

精确性分析：

  • 如果哦A是全0数组，始终输出是
  • 如果A是ξ-远离的

即这种情况下出错的概率小雨1/3

运行时间O(s)

证据引理：
如果一次测试以大于等于p的概率获得一个证据，那么s=2/p轮测试得到的证据概率大于等于2/3

对于一个判定问题L,其查询复杂性q(n)和近似参数ξ的性质测试算法是一个随机算法，其满足对于给定L的是一个实例x，最多进行q(|x|)次查询，并满足下述性质：

  • 如果x在L中，该算法最少2/3的概率返回“是”
  • 如果x是ξ远离L的，该算法以最少2/3的概率返回“否”

数据流中频繁元素—空间亚线性

大数据的数据流模型：

  • 数据是流动的，只能顺序的扫描一次或几次，复杂度不超过O(n)，需要能快速处理新到来的元素
  • 数据可能是无限的，内存有限的。大数据中内存规模 << 数据规模
  • 希望维护一个内存结果（概要）来给出相关性质的一个有效估计

数据流中可以计算什么？
例子：32,112,14,9,37,115,2
容易计算min,max,sum…使用单个寄存器s，直接更新

  • max:初始化s <- 0,对于每一个到来的新元素x, s <- max{x,s}
  • sum:初始化s <- 0,对于每一个到来的新元素x, s <- s+x
    s是可以合并的，多个流放到不同的s里，最后可以合并

频繁元素：元素出现多次，希望找到出现最频繁的元素(n表示不同元素的数量，m表示数据流当前接收到的元素的个数)
精确解：每个元素设置计数器，当内存只能提供k个计数器，k << n，问题就来了
近似算法(MG算法)：
对于元素x，如果已经给x分配计数器，计数器值自增；如果没有分配计数器，且计数器 < k，增加一个值为1的计数器；如果没有计数器，且计数器>=k，所有计数器减1，删除值为0的计数器。最后如果有x的计数器，返回其值，否则返回0,这种算法最后能输出最频繁的若干元素，但是输出值绝对低估了原本的重复值。

一个计数器x减少了几次？

  • 整个结构的权重（计数器的和）记作m’
  • 整个数据流的权重（全部元素的数量）是m
  • 每一个计数器降低的步骤减少k个计数，但是并未计，输入元素的此次出现，即k+1次未计入的元素出现。

最多有减少步骤（即估计值和真实值相差最多）：

当数据流的元素总数远大于这个值，x会是一个好的估计，错误的界限和k成反比，该算法有效的原因（zipf原则）
