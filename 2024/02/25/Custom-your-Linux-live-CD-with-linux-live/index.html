Custom your Linux live CD with linux-live

发表于 2024-02-25 更新于 2024-03-04 Disqus： 9.1k 8 分钟

Recently I was working on a Linux Bare Metal Recovery releated requirement. Users will be allowed to use a customed Linux live CD to run a temporary Linux system without making any changes to the storage, and then use the build-in tool to perform system recovery. So we need to find a way to custom our a Linux recovery live CD.

Due to I’m a newbie in such area, I searched a lot of linux live kit project from GitHub wishing to found a available solution. They are mostly some sorts of shell scripts to pack some specified Linux distro into a ISO file. I found the linux-live project by Tomas-M can pack any Linux system whose kernel supports squashfs , aufs and overlayfs. I just need to clone the project, and edit the config file in it to configure the path of our vmlinuz image and to select which paths need to be packed:

I’m using an Archlinux, and I use find /boot | grep vmlinuz to find where the vmlinuz image is:

The linux-live project is cloned to /opt and I don’t want it to be packed into live CD, so my config file finally looks like:

After that, run ./build. a temporary folder and two shell scripts will be generated to /tmp：


The /linux-data directory contains a squashfs image of the root filesystem and some bootfiles. The two scripts are used to generate ISO image or ZIP file from the /linux-data. I ran gen_linux_iso.sh and got an ISO image at /tmp/linux-x86_64.iso. I use VMware to create a new virtual machine, attach the ISO file to CD-ROM and chose to boot from CD-ROM. It works! A ugly boot background shows and the systems loads successfully!

However several days later, I found that the ISO cannot boot from UEFI(the previous vm is booted from BIOS). What’s worse, the linux-live project cannot build ISO image from aarch64 platform. I checked the build log and found it print error when executing initramfs_create. It fails because it uses some compiled binary utils like busybox and eject… in initramfs/static! So, this live kit won’t build definitely without doing any change to it! Now, to sum up, the live kit can only build ISO which works on x86_64 platform and support BIOS booting only, but we need to build ISO works on both x86_64 and aarch64 and can be boot from both UEFI and BIOS. I think, the binary utils format issue can solved by recompiling aarch64 platform, but the UEFI booting issue is really hard for me for I have no idea about how Linux boots from ISO!

I have written a blog How Linux Boots discussing how Linux boot from disk, but this time things got a little different cause we are trying to boot from a optical disk.
